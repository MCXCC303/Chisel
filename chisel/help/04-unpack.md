# 导入历史记录

导入是将之前导出的迁移包恢复到新环境的过程。

### 第一步：选择迁移包
指定之前生成的 `.tar.gz` 文件。同样支持输入路径或浏览选择。

### 第二步：指定目标目录
告诉 Chisel 新机器上的 Claude Code 配置在哪里。通常保持默认的 `~/.claude/` 即可。如果你选择的是一个普通目录（不是 `.claude`），Chisel 会自动在其中创建 `.claude/` 子目录，并把 `.claude.json` 放在合适的位置。

### 第三步：配置路径映射
迁移包中的每个项目都需要指定在新机器上的对应路径。Chisel 会根据包中的原始路径自动推断默认值：**它会把你原来的用户名替换为当前机器的用户名**，当你只是将数据迁移到一台主机名不同但用户名一致的机器上时，这个设定可以以最快的速度开始搭建目标项目。如果项目在新机器上的位置完全不同，可以手动修改。

### 第四步：开始解包
Chisel 会先做一次 dry-run 预览，告诉你将要复制多少文件，然后再实际写入。解包过程包括写入会话文件、合并项目元数据、追加命令历史等步骤。已有的数据不会被覆盖。
