# 常见问题

## 路径编码与碰撞

> 如果你是一位非 ASCII 字符语言（如中文、俄文）用户，你可能会发现在使用 Claude Code 时经常会出现历史记录混在一起的情况。

Claude Code 存在两套对特殊字符的编码：

| 位置            | 编码规则                           | 示例                                          |
|---------------|--------------------------------|---------------------------------------------|
| `claude.json` | 非 ASCII → `_`                  | `-home-alice-Applications-NextCloud-__-___` |
| 磁盘目录名         | 非 ASCII → `_`，随后 `_` 又被替换为 `-` | `-home-alice-Applications-NextCloud-------` |

这意味着同一个项目在 `.claude.json` 和磁盘上的编码名**不同**。我们认为这是 Claude Code 自身的设计缺陷。

这种不一致导致中文、特殊字符路径在编码时碰撞严重：不同路径可能碰撞，同一路径的 JSON 记录和磁盘目录也无法匹配。

Chisel 的扫描器会同时尝试两种编码来自动弥合这一差异，但并不保证能 100% 解决该问题。

### 案例

假设用户 `bob` 的电脑上有以下两个项目：

```
/home/bob/Documents/课程/药理学
/home/bob/Documents/课程/药剂学
```

编码后的结果：

| 原始路径                           | claude.json 中                  | 磁盘目录                           |
|--------------------------------|--------------------------------|--------------------------------|
| `/home/alice/Documents/烹饪/新东方` | `-home-alice-Documents-__-___` | `-home-alice-Documents-------` |
| `/home/alice/Documents/我要/金坷垃` | `-home-alice-Documents-__-___` | `-home-alice-Documents-------` |

两个完全不同的项目在 JSON 和磁盘上产生了**完全相同的编码名**。Claude Code 将两者的对话历史混存在同一个目录下，无法区分哪些对话属于哪个项目。

Chisel 会通过 `history.jsonl` 和会话 `cwd` 字段尽力将这些混在一起的会话分发回正确的项目。

**建议**：项目路径尽量使用纯 ASCII 路径，避免中文、emoji 等非 ASCII 字符。

## 包校验

### 缺少必要文件 / metadata.json 无效

**原因**：文件不是有效的 Chisel 迁移包，或包已损坏。  
**解决**：确认文件是 Chisel 生成的 `.tar.gz` 包，检查下载/传输是否完整。

### 包主版本不兼容

**原因**：打包时使用的 Chisel 版本与当前版本主版本号不同。  
**解决**：使用相同主版本的 Chisel 重新打包。

### 占位符 index 重复 / 会话文件缺失

**原因**：包内部元数据损坏。  
**解决**：重新打包，不要手动编辑 `.tar.gz` 内的 `metadata.json`。

## 打包

### 项目列表中看不到某些项目

**原因**：`~/.claude/projects/` 中不存在该项目的目录。扫描器只列出有实际会话文件的项目。  
**解决**：确认该项目确实使用过 Claude Code，检查 `~/.claude/projects/` 手动确认。

### 打包后文件过大

**原因**：选中的会话包含大量文件历史。  
**解决**：在会话选择步骤取消不需要的会话。

## 解包

### 解包后会话未出现

**原因**：目标机器上已存在相同 UUID 的会话，且内容分叉，被跳过。  
**解决**：检查解包报告中的「会话已跳过」数量。如需强制覆盖，手动删除目标机器上的对应文件后重新解包。

### 路径映射不正确

**原因**：占位符映射时路径配置错误。  
**解决**：在映射步骤检查每个项目的目标路径，手动修正。

### 跨机器迁移后路径问题

从一台机器迁移到另一台时，Chisel 会自动将 `/home/原用户名/后缀路径` 替换为 `/home/当前用户/后缀路径`
。如果两台机器用户名与后缀路径相同，默认路径将完全不变。如果不同，在解包的映射步骤中可以手动修改。

## 清除

### 删除了不该删除的记录

**原因**：清除操作不可逆，确认步骤时未仔细核对。  
**解决**：**无法恢复**。删除前务必使用预览功能确认每个会话的内容，或先打包备份。

### 清除后 claude.json 仍保留项目条目

**原因**：清除只会移除会话引用，不会删除整个项目条目。  
**解决**：空条目通常在 Claude Code 下次启动时自动清理，也可手动编辑 `~/.claude.json` 删除。

## 预览

### 预览内容显示乱码、空白或程序崩溃

**原因**：会话 jsonl 文件损坏，或包含特殊字符。  
**解决**：检查源文件是否完整，如可正常读取但预览异常，可能是 rich 标记解析问题。

### 预览加载缓慢

**原因**：会话内容非常大。  
**解决**：Chisel 已采用分批加载和内容裁剪。如仍有性能问题，跳过该会话预览即可。

## 程序问题

### 启动失败

**原因**：依赖未安装或 Python 版本过低。  
**解决**：

```bash
pip install -e .
python -m chisel
```

确认 Python >= 3.10，`textual` 和 `rich` 已安装。

### 文件选择器不工作

**原因**：系统未安装 `zenity` 或 `kdialog`。  
**解决**：手动输入路径，或安装 `zenity`：

```bash
sudo apt install zenity    # Debian/Ubuntu
sudo pacman -S zenity       # Arch
```
