# 错误排查

## 包校验失败

### `缺少必要文件: metadata.json`
**原因**：文件不是有效的 Chisel 迁移包，或包已损坏。  
**解决**：确认文件是 Chisel 生成的 `.tar.gz` 包，检查下载/传输是否完整。

### `包主版本 (N.x) 与当前 Chisel (M.x) 不兼容`
**原因**：打包时使用的 Chisel 版本与当前版本主版本号不同。主版本号变更意味着包格式有结构性变化。  
**解决**：使用与当前 Chisel 主版本相同的版本重新打包。如果确实需要跨版本迁移，可在旧版本机器上先解包到本地，再用新版本重新打包。

### `metadata.json 缺少字段: xxx`
**原因**：包元数据不完整，可能是旧版 Chisel 生成或手动修改过。  
**解决**：使用当前版本的 Chisel 重新打包。

### `占位符格式无效 / 占位符 index 重复`
**原因**：包内部元数据损坏。  
**解决**：重新打包。不要手动编辑 `.tar.gz` 内的 `metadata.json`。

### `占位符引用的会话文件缺失`
**原因**：打包过程不完整，部分会话文件未被包含。  
**解决**：重新执行打包。如果问题持续，检查源机器上 `~/.claude/projects/` 目录是否完整。

### `包中无会话数据文件`
**原因**：包是空的或不包含有效的会话数据。  
**解决**：确认打包时选择了至少一个包含会话的项目。

## 打包问题

### 项目列表中看不到某些项目
**原因**：`~/.claude/projects/` 中不存在该项目的目录。扫描器只列出有实际会话文件的项目。  
**解决**：确认该项目确实使用过 Claude Code。如果项目目录存在但编码名不匹配，可能是路径变更导致。检查 `~/.claude/projects/` 下手动确认。

### 打包后文件过大
**原因**：选中的会话包含大量文件历史（`file-history/`）。  
**解决**：在会话选择步骤取消不需要的会话。文件历史是可选的，如果只需要对话内容，可以跳过它（未来版本可能添加此选项）。

---

## 解包问题

### 解包后会话未出现
**原因**：目标机器上已存在相同 UUID 的会话，且内容分叉，被跳过。  
**解决**：检查解包报告中的「会话已跳过」数量。如果需要强制覆盖，需手动删除目标机器上的对应会话文件后重新解包。

### 版本不兼容警告
**原因**：包的 `chisel_version` 与当前 Chisel 版本不同。小版本差异通常兼容，但功能可能略有不同。  
**解决**：警告而非报错时可继续操作。如遇到实际功能异常，使用相同版本重新打包。

### 路径映射不正确
**原因**：占位符映射时路径配置错误。  
**解决**：在映射步骤检查每个项目的目标路径。如果项目在新机器上位置不同，手动修正。默认行为是替换用户名，如果用户名相同则路径不变。

---

## 清除问题

### 删除了不该删除的记录
**原因**：清除操作不可逆，确认步骤时未仔细核对。  
**解决**：**无法恢复**。删除前务必使用预览功能确认每个会话的内容。如果不确定，先打包备份再清除。

### 清除后 claude.json 仍保留项目条目
**原因**：清除只会移除已被删除的会话引用（`lastSessionId`），不会删除整个项目条目。空的 `projects` 条目通常在 Claude Code 下次启动时自动清理。  
**解决**：可手动编辑 `~/.claude.json`，删除对应的键。

---

## 预览问题

### 预览内容显示乱码、空白或程序崩溃
**原因**：会话 jsonl 文件损坏，或包含特殊字符。  
**解决**：检查源文件 `~/.claude/projects/<encoded>/<uuid>.jsonl` 是否完整。如果文件可正常读取但预览异常，可能是 rich 标记解析问题。

### 预览加载缓慢或卡顿
**原因**：会话内容非常大（数千条消息）。  
**解决**：Chisel 已采用分批加载（每批 30 条）和内容裁剪（最大 5000 行）。如果仍有性能问题，可考虑跳过该会话的预览，直接打包后在新机器上查看。

## 其他

### 程序启动失败
**原因**：依赖未安装或 Python 版本过低。  
**解决**：
```bash
pip install -e .
python -m chisel
```
确认 Python >= 3.10，`textual` 和 `rich` 已安装。

### 文件选择器不工作
**原因**：Chisel 基于 `zenity` 或 `kdialog` 来提供文件选择功能。确保系统上安装了至少一个服务。   
**解决**：手动输入路径，或安装 `zenity`：
```bash
sudo apt install zenity    # Debian/Ubuntu
sudo pacman -S zenity       # Arch
```
